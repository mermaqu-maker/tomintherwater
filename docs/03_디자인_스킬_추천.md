## ✅ 설치 상태 (2026-07-11)

리서치(02번) 기반 판단으로 **핵심 2개를 프로젝트에 설치** — `.claude/skills/`:
- **`image-first-grid-layout`** — 이미지가 무대가 되는 풀블리드 + 다크 그라디언트 + 절제된 오버레이. 우리 히어로·이미지 전면의 **핵심 디자인 시스템**.
- **`cinematic-gsap-lenis-motion-system`** — GSAP+Lenis 프리미엄 모션(스크롤 리빌·패럴랙스·호버·커서). **시네마틱 모션 레이어**.

**제외**: `cinematic-scroll-storytelling`(스티키 카드 스택 중심 단일 스크롤 내러티브용 — 갤러리 주도인 우리엔 모션 시스템이 더 범용), glass/webgl/laser 계열(순수 다크 미니멀과 불일치).
**이미 설치됨(중복 다운로드 불필요)**: `redesign-existing-projects`, `image-to-code`, `high-end-visual-design`, `frontend-design`, `minimalist-ui`.

## 0. 전제 — 우리가 이미 결정한 것 (02번 문서 기준)

- 배경 톤: **다크 `#1E1E1E`**, 무채색 UI, 색은 수중 이미지가 담당
- 내비: **4개 영문 메뉴 + sticky**
- 히어로: **풀블리드 1컷 + 절제된 타이포 + 진입 유도**
- 갤러리: **Masonry + hover 오버레이 + 감상 전용 라이트박스**
- 상세: **좌 이미지 / 우 설명, 스포트라이트형 서사**(카드 아님)
- 카피 톤: **서정적**("Dive Deeper" 류)
- 신뢰 레이어: About·상품에 경력/성과 절제 삽입

또한 `00_기존사이트분석_tom-inthewater.md`가 존재 — **이 작업은 신규 구축이 아니라 리뉴얼(redesign)**. 아래 추천에도 이 점을 반영.

## 1. 레퍼런스 → 재현 가능한 자료로 변환 (시안 착수 전, 선택)

02번 문서는 텍스트 기반 분석까지 끝난 상태. 실제 무드보드/토큰을 뽑기 전에 아래 스킬로 한 단계 더 정밀화할 수 있음.

| 스킬 | 용도 | 우리 프로젝트 적용 |
|---|---|---|
| `stitched-full-page-capture` | 레이지로딩/스크롤애니메이션 있는 페이지도 안정적으로 풀페이지 캡처 | Zena Holloway, Tim Tadder, Christy Lee Rogers처럼 스크롤 연출이 있는 사이트를 스크린샷으로 정밀 캡처 → 톤/간격 비교 자료로 사용 |
| `html-to-interaction-prompts` | HTML을 인터랙션/애니메이션 프롬프트로 추출 | 위 사이트들의 hover 오버레이, 갤러리 전환 방식을 그대로 뽑아 우리 스펙 프롬프트로 변환 가능 |
| `video-to-superprompt` | 스크린레코딩 영상을 재현용 초정밀 프롬프트로 변환 | Awwwards/Muzli 수상작 중 모션이 좋은 것을 녹화해두면, 이후 갤러리 전환·히어로 진입 모션 설계에 재활용 |

**우선순위: 낮음** — 이미 02번에서 정성적 분석은 끝났으므로, 시안 단계에서 특정 인터랙션(예: Zena Holloway의 타이틀 오버레이 방식)을 더 정밀하게 뜯어보고 싶을 때만 선택적으로 사용.

## 2. 무드/톤에 맞는 디자인 시스템 프리셋 (핵심)

라이브러리의 `web-design/` 카테고리는 대부분 "톤별 프리셋" 스킬. 우리 톤(다크·시네마틱·이미지 전면·절제된 UI)과 겹치는 후보:

| 스킬 | 설명 | 우리와의 적합도 |
|---|---|---|
| **`image-first-grid-layout`** | 풀블리드 사진 중심, 구조적 가이드라인, 앵커드 콘텐츠 블록, 절제된 테크니컬 오버레이 | ★★★ 갤러리·히어로가 전부 "이미지가 주인공"인 우리 방향과 정확히 일치 |
| **`cinematic-gsap-lenis-motion-system`** | GSAP+Lenis 기반 럭셔리 에디토리얼/크리에이티브 스튜디오 포트폴리오, Awwwards급 인터랙션 | ★★★ 02번 문서에서 참고 큐레이션으로 Awwwards/Muzli를 명시한 것과 직결. 사진작가 포트폴리오 특유의 고급스러운 스크롤 연출에 적합 |
| **`cinematic-scroll-storytelling`** | 스크롤 연동 스토리텔링, 스티키 카드 스택, 패럴랙스, 섹션별 서사 | ★★★ 프로젝트 상세의 "스포트라이트 서사"(Christy Lee Rogers 참고점)를 구현할 때 직접 활용 가능 |
| `editorial-tech` | 매거진 편집 구성 + 절제된 액센트, 비대칭 그리드 | ★★ 텍스트 최소·타이포 위계 단순 방향과 부분적으로 맞음. 다만 "테크" 느낌이 과하면 우리 서정적 톤과 어긋날 수 있어 검토 필요 |
| `minimalist-ui` | 웜 모노크롬, 타이포 대비, 플랫 벤토 그리드 | ★★ 내비·UI 절제 방향엔 맞으나 "벤토 그리드"는 Masonry 갤러리와 결이 다름 — 참고만 |
| `dark-glass-clean-layout` / `glass-dark-ui` | 프로스티드 글래스, 블러 패널 | ★ 우리는 글래스모피즘을 쓰지 않는 순수 다크 미니멀이라 비적합. 참고 목록에서 제외 권장 |

**우선순위: 높음** — `image-first-grid-layout` + `cinematic-gsap-lenis-motion-system`(또는 `cinematic-scroll-storytelling`) 조합이 02번 문서의 결론과 가장 가깝다고 판단.

## 3. 모션/디테일 마감

| 스킬 | 용도 | 우리 프로젝트 적용 |
|---|---|---|
| `staggered-word-reveal` / `masked-reveal` | 단어 단위로 순차적으로 떠오르는 텍스트 리빌 | 히어로의 "한 줄 서정적 카피"나 프로젝트 상세 타이틀에 적용하면 Erik Johansson류의 몰입감 진입 연출과 맞닿음 |
| `css-alpha-masking` | 이미지 가장자리 페이드 | 풀블리드 히어로 이미지가 하단/사이드로 자연스럽게 사라지는 처리에 사용 |
| `progressive-blur` | 레이어드 backdrop-blur | 갤러리 hover 오버레이 진입/이탈 시 부드러운 블러 트랜지션에 활용 가능 (과하면 우리 "절제된" 톤과 충돌하니 소량만) |
| `animation-systems` | Stripe/Linear/Apple급 모션 원칙(이징, 접근성 reduced-motion 포함) | 개별 이펙트보다 상위 원칙 — 전체 모션 톤을 잡을 때 기준점으로 참고 |

## 4. 품질 관리 — "AI스러운 뻔한 디자인" 회피

사진작가 포트폴리오는 특히 "이미지가 주인공, UI는 존재감 없이"가 핵심이라 이 계열 스킬이 유효.

| 스킬 | 용도 |
|---|---|
| `redesign-existing-projects` | **기존 사이트(tom-inthewater) 감사 → 제네릭 패턴 식별 → 프리미엄 기준 적용, 기능은 유지.** 이 프로젝트가 신규가 아니라 리뉴얼이라는 점에서 4개 항목 중 컨텍스트상 가장 직접적으로 맞는 스킬 |
| `frontend-design` / `high-end-visual-design` | 생성 코드가 "AI 템플릿" 티 나는 걸 방지하는 규칙 세트. 절제된 다크 미니멀일수록 사소한 디폴트(그림자, 라운드 코너 등)에서 티가 나기 쉬움 |
| `image-to-code` | 레퍼런스를 참고해 디자인 이미지를 먼저 만들고 분석한 뒤 코드로 정밀 구현 — "레퍼런스 조사 완료 → 시안 제작" 단계에 정확히 대응 |

## 6. 종합 추천 순서 (실행 관점)

1. `redesign-existing-projects`로 기존 tom-inthewater 사이트를 감사 기준 삼아 "버릴 것/살릴 것" 재확인
2. `image-first-grid-layout`을 베이스 디자인 시스템으로 채택
3. `image-to-code`로 무드보드 이미지를 먼저 뽑고, 02번 문서의 레퍼런스(Zena Holloway, Tim Tadder, Christy Lee Rogers)와 비교 검토
4. 히어로·상세 서사에는 `cinematic-scroll-storytelling` 또는 `cinematic-gsap-lenis-motion-system`으로 스크롤 연출 설계
5. 타이포 리빌은 `staggered-word-reveal`, 이미지 처리(fade/blur)는 `css-alpha-masking`·`progressive-blur`로 마감
6. 전체 결과물을 `frontend-design`/`high-end-visual-design` 기준으로 마지막 점검
