## 0. 목표 (이 사이트가 해내야 하는 것)

기존 Adobe Portfolio 사이트는 "감상"에서 끝났다. 새 사이트의 핵심 미션은 **감상 → 신뢰 → 카카오톡 문의**로 이어지는 동선.

| 우선순위 | 목표 | 성공 지표 |
|---|---|---|
| P0 | 카카오톡 채널 문의 전환 | 채널 CTA 클릭 수 |
| P0 | 몰입감 있는 포트폴리오 감상 | 프로젝트 상세 체류시간 / 이탈률 |

| P1 | 관리자 직접 콘텐츠 관리 (빌더 탈피) | 코드 배포 없이 프로젝트 CRUD |

## 1. 정보구조 (IA)

**상단 메뉴 4개**로 단순화. 문의는 별도 페이지가 아니라 사이트 전역 카카오톡 CTA.

```
/                        홈 (히어로 + 대표작 + 카카오 CTA)
/about                   About (신뢰 콘텐츠)
/gallery                 갤러리 허브
  ├─ In the Water        수중 (기본 포트폴리오)
  └─ On the Water        지상/수면/스튜디오
/gallery/[slug]          프로젝트 상세 (두 카테고리 공통 템플릿)
/products                상품 소개 (촬영 상품/패키지 + 의상 대여 진입 링크)
/costumes                의상 대여 카탈로그 (메뉴 밖 · 상품소개/직접링크로만 진입)
/reviews                 촬영 후기 (고객 후기, 관리자 삭제 가능)
```

### 핵심 변경 (기존/이전안 대비)
- **메뉴 4개 재편**: About · 갤러리 · 상품 소개 · 촬영 후기.
- **갤러리 = 2 카테고리**: In the Water(수중) / On the Water(지상). 기존 **Event · Commercial은 별도 카테고리를 없애고 두 카테고리 안의 태그로 편입**. (예: 수중 이벤트 촬영 = In the Water + `event` 태그)
- **문의 폼 폐지**: 구조화 폼/`/contact` 없음. 대신 **카카오톡 채널로 연결되는 CTA를 사이트 곳곳(홈·상세·상품·About 등)에 배치**.
- **별도 어드민 페이지 없음**: 관리자로 로그인하면 **각 페이지에서 인라인으로 직접 편집**.
  - 갤러리/프로젝트 페이지 → 프로젝트 **추가·수정·삭제**.
  - 촬영 후기 페이지 → 스팸/부적절 후기 **삭제**(모더레이션).

## 2. 페이지별 설계

### 2.0 전역 UI (모든 페이지 공통)
- **브랜드 표기**: `TOM`(단독) / `TOM IN THE WATER`(풀네임, 헤더 로고·푸터).
- **헤더**: 로고 + **영문 메뉴 4개** — **About · Gallery · Course · Reviews**. 헤더에는 카카오 CTA 없음.
  - **Gallery hover → 드롭다운**: In the Water(수중 작업) / On the Water(지상 작업).
  - **Course hover → 드롭다운(3개 그룹, 숫자 없음)**:
    - **Essential** — 기본 촬영(Basic · Premium)
    - **Special** — 특별 촬영(Blue Lagoon · Moonshore) *(상시 촬영 아님)*
    - **Group & Commercial** — 그룹 · 이벤트 · 상업
  - 서브메뉴 클릭 → 상품 페이지에서 **그 그룹의 상품만** 표시(아래 2.5).
- **우하단 플로팅 버튼(상시 노출, 미니멀)**: 위 = **카카오톡 문의** 아이콘, 아래 = **TOP(맨 위로)** 아이콘. 사이트 전역 고정.
- **푸터**: 브랜드 + **아이콘 2개만**(작은 Instagram 아이콘 + 말풍선 아이콘=카카오톡). 텍스트 메뉴 나열 없음.

### 2.0-a 프로젝트 카드 공통 규칙
- **캡션을 이미지 아래에 붙이지 않음.** 사진만 오밀조밀 배치해 이미지가 돋보이게.
- **마우스 오버 시**: 사진 위에 반투명 블랙 오버레이 + 그 위에 프로젝트명/연도 텍스트 레이오버.
- **한 줄 최대 2개**(갤러리·대표작). 관련작(You may also like)만 예외적으로 더 작게 3열 유지.

### 2.1 홈 `/`
- **콘텐츠는 히어로 + 대표 작업(Featured work) 두 섹션만.** (상품/후기/About 티저·CTA 밴드 없음 — 미니멀)
- 풀블리드 히어로: 배경 영상 or 대표 수중 이미지 + 서정적 한 줄 카피 (*"A journey underwater to meet another Me."*). **히어로에 카카오 CTA 넣지 않음** — 문의는 우하단 플로팅으로만.
- 대표 프로젝트 그리드 (수중/지상 믹스) → 갤러리 전체 보기.
- **관리자 Featured 관리(홈에서 인라인)**: 관리자 로그인 시,
  - 각 대표작에 **`−` 아이콘**으로 Featured에서 제외("삭제"가 아니라 **대표작에서만 빼는 것** — 프로젝트 자체는 유지).
  - **`＋` 타일**로 신규 추가 → **프로젝트 선택 팝업**(등록된 프로젝트 목록을 **분류(수중/지상)·연도로 필터**해서 고름).

### 2.2 About `/about`
기존의 빈약함을 신뢰 콘텐츠로 보강:
- 작가 소개 (서정적 콘셉트 문장 유지)
- 경력 / 주요 클라이언트 / 수상 / 장비 / 촬영 가능 범위
- 연락: Instagram [@tom_freedive](https://www.instagram.com/tom_freedive/), **카카오톡 채널 CTA**

### 2.3 갤러리 `/gallery` + 카테고리
- 허브에서 In the Water / On the Water 진입 (또는 한 페이지 + 카테고리 토글)
- 카테고리 페이지 = 프로젝트 썸네일 **Masonry 그리드**(가로·세로 사진 혼합을 원본 비율 그대로, 2열). 캡션 없이 hover 오버레이(2.0-a).
- **필터(영문 통일) — 두 축, 모두 "관리자가 등록 시 넣는 값"에서 나옴**:
  - **성격(Type) 토글**: `ALL / EDITORIAL / EVENT / COMMERCIAL` — 프로젝트 등록 시 관리자가 지정하는 **type 태그** 기준(복수 지정 가능).
    - Editorial = 작가 개인·작품·컨셉 화보(포트폴리오 기본), Event = 이벤트 현장 촬영, Commercial = 클라이언트·상업 작업.
  - **패싯 드롭다운(값 선택)**: `PLACE ▾ / YEAR ▾` — **미리 정하지 않고 등록된 프로젝트에서 자동 생성**.
    - PLACE = 프로젝트 `location`의 distinct 값, YEAR = `shot_date`의 distinct 연도. 새 프로젝트 추가 시 자동 반영.
    - 장소 중복 방지를 위해 등록 폼에서 **기존 장소 자동완성 + 신규 추가** 방식(권장). 값이 2개 이상일 때만 필터 노출.

> **혼합 방향 레이아웃**: 가로/세로가 섞인 사진을 자동 배치. 구현은 **`react-photo-album`**(masonry/justified-rows, 반응형, next/image 연동) + 라이트박스는 **`yet-another-react-lightbox`**(전체화면·좌우 화살표·키보드, 2.4 스펙과 일치) 추천. 각 이미지 `width/height`를 저장해 서버 렌더 시 레이아웃 시프트(CLS) 방지.

#### 빈 상태(Empty state) 스펙 — 데이터가 없을 때
공개 프로젝트가 아직 없을 수 있으므로(작가가 이후 등록), 모든 갤러리 표면은 빈 상태를 우아하게 처리한다. 깨진 빈 그리드 금지.
- **카테고리 전체가 0건**: 그리드 대신 중앙 정렬 안내 — eyebrow(`GALLERY`) + `아직 공개된 작업이 없습니다.` 한 줄(다크 톤, 미니멀). 필터바는 숨김.
- **필터 결과만 0건**(장소/연도/성격 조합): `해당 조건의 작업이 없습니다.` + **[필터 초기화]** 링크. 필터바는 유지.
- **PLACE ▾ / YEAR ▾ 드롭다운**: 해당 카테고리에서 **값이 1개 이상 있을 때만 렌더**(뷰에서 distinct 집계). 값이 없으면 그 칩 자체를 숨김 → 미리 정하지 않아도 데이터에서 자동 생성.
- **성격 토글(EDITORIAL/EVENT/COMMERCIAL)**: 고정 항목이라 항상 노출. 매칭 0건이면 필터 결과 빈 상태로.
- **홈 Featured 0건**: 대표작 섹션 자체를 숨김(또는 최소 안내). 공개 방문자에게 빈 그리드 노출 금지.
- **프로젝트에 커버 이미지 없음**: 다크 그라디언트 플레이스홀더 + 제목만으로 카드 렌더(그리드가 깨지지 않게).

### 2.4 프로젝트 상세 `/gallery/[slug]`
- **상단 = 대표 이미지 + 설명 2단 레이아웃**(히어로를 화면 꽉 채워 자르지 않음):
  - **좌측**: 대표 이미지를 **잘리지 않게 원본 비율 그대로**(반응형, contain). 모바일에선 상하로 스택.
  - **우측**: 제목 · **촬영 일시** · breadcrumb(수중/지상) · 서정적 캡션 · **표준 크레딧**(장소/협업자/장비) · 태그.
    - 장소는 크레딧에 이미 있으므로 **"장소 태그"는 두지 않음**(event 등 성격 태그만).
- **하단 = 프로젝트 이미지 타일**: 가로/세로 혼합 **Masonry(2열)**, 원본 비율 유지. 각 타일 클릭 → 전체화면 라이트박스.
- **라이트박스(사진 확대) = 감상 전용**: 화면 꽉 채워 확대, 좌우 화살표(←/→)로 프로젝트 내 사진 넘김, 카운터, 닫기(Esc/X). **좋아요·댓글 없음** — 작가 의도상 작품을 온전히 감상만.
- **좋아요 + 댓글은 프로젝트 단위**(아래 2.4-a). 상세 페이지 하단에 배치.
- prev/next(이전·다음 프로젝트) + **"You may also like" 3개(작게)**.
  - **추천 기준**: ① 같은 카테고리(수중/지상) ② 공유 태그 많은 순 ③ 부족하면 최신순. + 관리자 수동 고정 옵션.
- **관리자 로그인 시**: 편집/삭제, 신규 추가.

### 2.4-a 프로젝트 단위 좋아요 · 댓글 (사진 상세 아님)
좋아요·댓글은 개별 사진이 아니라 **프로젝트 1건 단위**로만. 상세 페이지 하단(이미지 타일 아래)에 위치.
- **좋아요**: 로그인·제한 없이 자유롭게 누름(중복 방지 없음, 카운트만 증가).
- **댓글**: **닉네임 + 비밀번호** 입력 후 작성(텍스트만 — 사진 첨부 없음).
  - **수정/삭제**: 본인이 기입한 **비밀번호를 다시 입력해야** 가능.
  - **관리자 로그인 시**: 비밀번호 없이 아무 댓글이나 삭제(모더레이션).

### 2.5 상품 소개 `/products` (= Course)
촬영 상품/패키지 소개 — 신뢰·전환의 핵심. **설명 영역을 리치하게**(대형 이미지 + 강조된 가격/포함사항).

**3개 그룹으로 분리**(Course 드롭다운과 동일):
- **Essential(기본 촬영)**: ① BASIC · ② PREMIUM
- **Special(특별 촬영)**: ③ BLUE LAGOON HALL · ④ SIGNATURE(Moonshore)
- **Group & Commercial**: ⑤ GROUP/EVENT/COMMERCIAL

서브메뉴/탭 선택 시 **그 그룹 상품만** 표시. **Essential ↔ Special은 하단에 크로스 추천 카드**로 서로 연결("이런 촬영은 어떠세요?" → 클릭 시 상대 그룹으로 전환). Group & Commercial은 크로스 없음.

1. **BASIC COURSE — 깔끔하고 직관적인 수중 프로필**
   - 수영복 / 슈트 / 머메이드테일 · 1인/2인(커플·우정) · 의상 개인 준비(슈트·테일 대여 가능, 유료)
   - **25만 원** · 헬퍼 옵션 +5만 원 · 포프라자/딥스테이션/MS다이빙풀 중 택일
2. **PREMIUM COURSE — 환상적인 드레스, 고난도의 연출**
   - 드레스 / 셔츠 / 코스프레 · 1인/2인(커플·우정) · 의상 개인 준비(대여 가능, 유료) · 안전·완벽 촬영 헬퍼
   - **35만 원** · 헬퍼 기본 포함 · 포프라자/딥스테이션/MS다이빙풀 중 택일
3. **BLUE LAGOON HALL — 여름 한정! 웨이브파크 블루라군홀 촬영**
   - 야외 수중 환경 · 2인/3인 전용 · 의상 개인 준비(수영복·슈트·머메이드) · 의상 대여 가능(유료)
   - **2인 진행 1인 15만 원 / 3인 진행 1인 10만 원**
4. **SIGNATURE CONCEPT with MOONSHORE** *(준비 중)*
   - 언더워터웨어 브랜드 MOONSHORE 협업 · 테마별 전용 의상·소품·세팅 제공 · **COMING SOON**
5. **GROUP / EVENT & COMMERCIAL**
   - 단체(3인 이상) / 행사 / 상업 촬영 · `@tom_freedive` 계정으로 별도 문의

- **상품별 촬영 예시 사진**: 상품 카드 이미지(또는 "촬영 예시 N장 보기" 링크) 클릭 → 그 상품 예시 사진을 **전체화면 라이트박스**(좌우 넘김)로 감상. 상품마다 별도 예시 이미지 세트.
- **의상 대여 진입 링크/블록** 포함 → `/costumes`.
- **페이지 하단 문의 블록**: "촬영 문의는 아래로 주세요" + **[문의하기] 버튼 → 카카오톡 채널**. (기존 "카카오톡으로" 저채도 안내 문구는 제거)

### 2.7 의상 대여 카탈로그 `/costumes`  *(상단 메뉴에 없음)*
드레스·인어테일 등 유료 대여 의상을 필터로 탐색하는 독립 페이지. 대여의 핵심은 아니라 헤더 메뉴에는 노출하지 않고, **두 경로로만 진입**:
1. `/products`(상품 소개) 안의 진입 링크 — 촬영 희망자가 대여 가능 의상을 확인.
2. **직접 URL** — 카카오톡 채널 상담 중 바로 공유하는 링크.

- **필터**:
  - **유형**: 수영복 / 슈트 / 드레스 / 치마 / 머메이드 테일 / 컨셉 의상 / 소품.
  - **사이즈**: 44 / 55 / 66 / 77 / Free.
  - **컬러**.
  - **촬영 종류(가능 코스)**: 의상 난이도에 따라 촬영 종류가 갈림 → 이 의상을 **어떤 코스에서 입을 수 있는지** 필터. 예: 긴 드레스 = PREMIUM 이상만.
  - **가격 필터는 없음**(가격은 카드에 표시만; 1만~5만원 1만원 단위).
- 의상 카드 그리드: 대표 이미지 + 이름 + 유형/사이즈/컬러 + 가능 코스 + 대여가.
- **카카오 문의 안내/버튼은 페이지에서 제거**(전역 플로팅으로만 문의). 예약/결제 없음.
- **관리자 로그인 시**: 의상 인라인 추가·수정·삭제.

### 2.6 촬영 후기 `/reviews`
- 평소에는 **작성된 후기 목록만** 노출(작성자·별점·**촬영 상품**·본문·선택 이미지·날짜).
- **"후기 작성하기" 버튼 → 팝업(모달)** 으로 큰 작성란 표시. 상시 폼 노출 아님.
- **작성 폼 항목**: 닉네임 · **촬영 상품 선택**(BASIC/PREMIUM/BLUE LAGOON/MOONSHORE/GROUP) · **별점** · 후기 내용 · (선택)사진 · 캡차.
  - **별점 입력 방식**: 별 5개를 놓고 **마우스 오버 시 미리보기 채움 → 클릭한 별까지 확정**(1~5점) 인터랙티브 방식.
- **목록 상품 필터**: 상단에 촬영 상품별 필터 칩(전체/BASIC/PREMIUM/BLUE LAGOON/MOONSHORE/GROUP)으로 후기를 걸러 봄. 각 후기 카드에 상품 배지 표시.
- **방문자 공개 작성**: 로그인 없이 누구나. 스팸 방지(캡차/레이트리밋).
- **목록 페이징 = "더 보기(load-more)" 버튼** 방식으로 결정. (무한 스크롤은 푸터·플로팅 문의 접근 저해로 제외, 숫자 페이징은 과함. 후기 수백 개로 커지면 페이징 재검토.)
- **관리자 로그인 시**: 스팸/부적절 후기 **삭제** 버튼 노출(모더레이션).

### 2.8 관리자 인증 · 편집 플로우
별도 어드민 대시보드 없이 **로그인하면 사이트 전역에서 인라인 편집**하는 구조.
- **진입**: 메뉴에 없는 **직접 URL `/admin`**(또는 `/login`). 방문자에겐 노출 안 함.
- **인증**: **Supabase Auth 이메일 + 비밀번호**(1인 소유자 계정).
- **비밀번호 재설정**(Supabase 기본 제공, 구현 간단):
  - 로그인 화면 "비밀번호를 잊으셨나요?" → 이메일 입력 → `resetPasswordForEmail(email)` → 재설정 링크 발송.
  - 링크 클릭 → `/admin/reset`(복구 세션 자동) → 새 비밀번호 입력 → `updateUser({ password })`.
- **로그인 상태**: 하단 **관리자 바** 고정(● 관리자 · TOM · 편집 모드 ON · 로그아웃). 세션 유지.
- **편집 UI(전역 인라인, 모달)**:
  - 프로젝트: `＋ 추가` / `✎ 수정`(모달: 제목·촬영일시·카테고리·공개여부·캡션·크레딧·태그·이미지 업로드/순서) / `🗑 삭제`(확인 다이얼로그).
  - 홈 Featured: `−`(목록에서만 제외, 확인창 없음) / `＋`(프로젝트 선택 팝업).
  - 상품·의상: `✎ 수정`(의상 모달: 이름·유형·사이즈·컬러·가능코스·대여가·이미지) / `🗑 삭제`.
  - 후기·사진 댓글: `🗑 삭제`(모더레이션, 확인).
- **로그아웃 시** 편집 버튼 전부 사라짐. 방문자에겐 처음부터 안 보임.
